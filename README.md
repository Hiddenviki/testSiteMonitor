# тестовое задание site monitor

Программа на Python для мониторинга доступности сайтов из перечня и формирования суточного отчёта (за выбранный день).

## Реализованный функционал

- Проверка доступности сайтов из `sites.json` с периодом опроса.
- Проверка достоверности результата:
  - ресурс считается **доступным**, если HTTP‑статус \(2xx–3xx\) и в ответе находится контрольная строка `expect`;
  - иначе фиксируется ошибка (таймаут/ошибка сети/плохой статус/несовпадение контента).
- При отказе сохраняется лог: **«Наименование организации – время отказа сайта – наименование ошибки»**.
- После восстановления сохраняется лог: **«Наименование организации – время восстановление сайта»**.
- Отчёт за сутки: **домен**, **uptime, % (до 2 знаков)**, **суммарный простой** за день.

## Структура

- `sites.json` — перечень 5 организаций (банки/страхование) и параметры проверки.
- `data/monitor.sqlite3` — база SQLite (создаётся автоматически): результаты проверок и инциденты.
- `logs/monitor.log` — файл логов (создаётся автоматически).

## Установка

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Запуск мониторинга

Непрерывно (период опроса задаётся в `sites.json`):

```bash
python -m site_monitor monitor
```

Одна итерация (удобно для быстрой проверки):

```bash
python -m site_monitor monitor --once
```

## Формирование отчёта за сутки

```bash
python -m site_monitor report --date 2026-03-26
```

Опционально сохранить CSV:

```bash
python -m site_monitor report --date 2026-03-26 --csv reports/2026-03-26.csv
```

## Примечания

- Часовой пояс берётся из `sites.json` (по умолчанию `Europe/Moscow`).
- Uptime считается как (время суток - downtime) / время суток * 100 и выводится с точностью до 2 знаков.
